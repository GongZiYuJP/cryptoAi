## 运行步骤

### 步骤1: 检查依赖包

```bash
python check_dependencies.py
```

这会检查所有必需的依赖包是否已安装。

### 步骤2: 安装依赖（如果缺少）

如果步骤1显示缺少依赖包，运行：

```bash
# 方法1: 使用requirements.txt（推荐）
pip install -r requirements.txt

# 方法2: 直接安装
pip install ccxt pandas numpy requests ta

# 方法3: 使用自动安装脚本
python install_and_test.py
```

### 步骤3: 验证代码

```bash
python verify_code.py
```

这会验证代码的语法和结构。

### 步骤4: 配置系统

1. 检查是否有 `config.py` 文件
2. 如果没有，创建配置文件：

```python
# config.py
EXCHANGE_CONFIG = {
    'exchange': 'binance',
    'apiKey': '您的API_KEY',
    'secret': '您的SECRET'
}

TELEGRAM_CONFIG = {
    'token': '您的BOT_TOKEN',
    'chat_id': '您的CHAT_ID'
}

TRADING_CONFIG = {
    'portfolio_size': 1000.0,
    'risk_per_trade': 0.015,
    'symbol': 'ETH/USDT:USDT',
    'check_interval': 300
}
```

### 步骤5: 测试Telegram连接（可选）

```bash
python test_telegram.py
```

### 步骤6: 运行交易系统

```bash
python run_trader.py
```

或者直接运行主程序：

```bash
python cryptoAi2.py
```

## 依赖包列表

必需的包：
- `ccxt` - 交易所API库
- `pandas` - 数据处理
- `numpy` - 数值计算
- `requests` - HTTP请求

## 常见问题

### Q: 导入错误 "ModuleNotFoundError: No module named 'xxx'"

**A**: 运行 `pip install -r requirements.txt` 安装所有依赖

### Q: Windows编码错误

**A**: 代码已自动处理Windows编码问题，如果仍有问题，确保终端支持UTF-8

### Q: 交易所连接失败

**A**: 
1. 检查API密钥是否正确
2. 检查网络连接
3. 代码会自动降级到公共API模式（仅读取数据）

### Q: Telegram通知收不到

**A**: 
1. 运行 `python test_telegram.py` 测试连接
2. 确保已向Bot发送过至少一条消息
3. 检查Bot Token和Chat ID是否正确
